
# 4.2实现fork函数

1. 系统调用表（sys_call_table），包含了所有系统调用函数的指针。sys.h
2. 修改0x80系统调用的中断逻辑，从系统调用表中按调用号执行对应的函数——system.S
3. 添加fork系统调用的实现函数（sys_fork）——system.S
	1. 查找空进程表项nr（find_empty_process）
	2. 复制nr进程（copy_process）
	3. 返回结果
4. 使用宏定义通用系统调用函数，利用NR+name的方式动态选择调用的编号—— include/unistd.h
5. 实现进程复制函数（copy_process）
	1. 获取一个新的页，任务表（task）中登记
	2. 复制当前进程的 task_struct
	3. 设置子进程pid和指向父进程的指针
	4. 复制TSS
	5. 复制父进程内存空间（copy_mem）
	6. 设置新的TSS、LDT描述符
	7. 返回
6. 实现复制进程内存空间（copy_mem）
	1. 取当前进程的代码&数据段基址
	2. 设置新进程的LDT&数据的基址
	3. 调用复制页表函数（copy_page_tables）从当前进程复制页表到新进程
	4. 返回
7. 实现页表复制函数
	1. 定位页目录，通过右移 20 位并清空低 2 位，找到全局唯一的页目录表
	2. 遍历父进程页目录，如果页目录存在，则申请一个新物理页存放其对应的页表

8. 实现页保护
	1. 添加页中断汇编代码（mm\page.s）
	2. 实现页写保护函数（void do_wp_page(...)）
		1. 判断是否为init进程
		2. 计算对应的页表项位置
		3. 判断是否计时器为1，为1说明是页面唯一使用者
		4. 否则申请新空间，修改页表项指向新空间，原页表项引用减一
		5. 返回
9. 实现进程调度
	1. 添加进程状态宏，在PCB中添加相关参数
	2. 设计进程调度函数（schedule()）
		1. 从当前进程选取count最大的进程，设置为运行态
		2. 如果当前所有就绪进程的时间片都用完了，设置新count时间片
	3. 执行切换
